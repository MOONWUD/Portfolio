Objective

기존의 웹페이지는 귀여운 캐릭터를 잘 활용하지 못하고 단조로운 느낌을 줍니다.
래핑찰리의 귀여운 캐릭터를 활용한 통통 튀는 디자인과
반응형 웹페이지를 구현해 퍼블리싱하는 것을 목표로 선정했습니다.

Problem &
Solution

  • Problem 01

    header 배경에 blur처리를 하고 싶어 filter를 활용하면
    배경이 아니라 전체에 blur가 씌워짐 !

    새로운 bg 영역을 만들어 blur처리 했지만 자연스럽지 않음

    잠시 잊고 있었던 backdrop filter의 blur를 활용하였더니 원하는 느낌을 줘 해결 !

  • Problem 02

    Main 이미지 위에 텍스트를 고정하기 위해서
    position을 활용하니 알 수 없는 이유로 레이아웃이 무너진다 !

    각 이미지와 텍스트를 감싸는 요소를 만들어서 처리했지만 여전히 무너짐

    grid를 활용해 이미지와 텍스트를 같은 선상에 있도록 만들어 해결 !

  • Problem 03

    자연스러운 가로 드래그가 이어지지 않는 문제 발생 !

    여러 클릭 드래그를 구현했지만, 실패했음
    아마 내가 구성한 css에 근본적인 문제가 있을 것으로 추정

    클릭 드래그에서 벗어나 스와이퍼를 이용하여 해결 !

  • Problem 04

    pc header와 달리 mo header에는 스와이퍼를 넣으려 하니
    전체적인 레이아웃이 무너짐

    jQuery를 활용한 미디어쿼리를 사용하니 새로고침이 필수이며,
    자연스럽지 않은 움직임이 문제임

    pc와 mo을 따로 만들어서 해결
    ( 근본적으로 mo를 먼저 작업하지 않은 것이 문제라 판단 )

  • Problem 05

    작은 사이즈의 기기(13인치 노트북)로 접속했을 때
    text nowrap이 활성화되지 않고 줄바꿈 되는 문제 발생 !

    요소의 전체적인 가로 사이즈를 늘려 해결

  • Problem 06

    walk section과 pogle section에 스와이퍼를
    버튼이 있을 때만 활성화 시키려고 prependTo()와 appendTo()를 활용하여
    구현하였지만 움직임이 부드럽지 않음 !

    jQuery를 활용한 미디어쿼리를 사용하니 새로고침이 필수이며,
    자연스럽지 않은 움직임이 문제임

    java의 animate를 이용해 marginLeft로 움직이고,
    버튼이 2번만 활성화되게 설정 !

Review

  • 이번 프로젝트는 작업일지와 변수, 가이드라인을 더욱 꼼꼼하게 작업했다.
    새로운 프로젝트에 들어갈 때마다 치밀해지는 나를 발견할 수 있었다.
    저번 작업 초기에 고려하지 못한 부분이 나중에
    큰 걸림돌이 되었던 것을 기록, 기억해 매번 더 나은 나로 성장하고 있다.
  • 테블릿을 디자인하지 않았다는 내용 추가 !
  • 다만 아쉬운 점은 반응형 웹을 구현할 때 모바일 먼저 작업하지 못한 것이다.
    하지만 이번 작업을 통해서 모바일을 먼저 작업하는 이유를 명확하게 알았고,
    다음번에는 같은 실수를 반복하지 않을 것이다.
  • 반응형을 작업하면서 가장 힘들었던 점은
    변화하는 사이즈에 대응하여 단위 고려해야한다는 것이다.
    세세하게 분기점을 둬 작업할 순 없는 일이다.
    유연한 단위를 사용하는 것이 가장 중요하다 느꼈다.
  • 이번 프로젝트는 애정을 담아 즐겁게 작업했다.
    좋아하는 일을 할 때 능률이 월등히 높다.
    나에게 떨어진 과업이 설령 좋지 않다하더라도
    그 순간만큼은 사랑에 빠진 것처럼 몰입해야겠다 생각했다.
