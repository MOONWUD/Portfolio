Problem &
Solution

  • Problem 01

    main video 영역에 반응형 구현시 비율로 인한 어려움을 겪음”

    behavior: 'smooth'를 사용해 스크롤을 부드럽게 적용했으나 여전히 버벅거림
    fullpage.js를 활용해보았지만, 다른 이벤트가 적용되지 않아 실패했음

    youtube 영상 자체에 width 값을 크게 주어 영역을 넘는 부분에 overflow hidden을 사용

  • Problem 02

    가로스크롤 영역부분 반응형시 사진이 넘치는 현상이 발생

    transform을 이용해서 두 section의 겹치는 요소가 viewport같은 위치에 있도록 구현했지만
    여전히 어색하여 요소가 스르륵 이동하게 구현하는 방법을 고려

    모바일에 경우 가로 스크롤을 보는 경우가 드물어 세로로 된 형태로 사진을 돋보이게 만듦

  • Problem 03

    GSAP ScrollTrigger를 이용했을때 속도와 타이밍 문제가 발생

    같은 버튼 스타일을 사용하였지만, 하나만 이상하게 제대로 작동하지 않아 원인을 찾음
    a태그를 감싸고 있는 부모요소에 margin: 0 auto;를 줘 가운데 정렬한 것이 원인.
    문제가 되는 버튼에만 margin값을 초기화

    duration과 scrub 값을 조절하고 start와 end 값을 지정해 해결, pc 버전에서만 gsap를 적용

  • Problem 04

    product-slideShow를 setInterval로 구현 시 점점 빨라지는 현상 발생

    미디어쿼리를 활용해 맞추려고 했으나
    다양한 PC사이즈를 감당할 수 없었음

    css의 animation을 이용하여 transform: translateX() 값 조정을 통해 구현 완료

  • Problem 05

    색상선택시 사진이 변경되는 JS 구현시 어려움을 겪음

    미디어쿼리를 활용해 맞추려고 했으나
    다양한 PC사이즈를 감당할 수 없었음

    backgroundimage를 이용해 addClass와 removeClass를 사용하여 구현 완료

  • Problem 06

    배경 쉐입 변경 이벤트 구현시, 요소 넘침 + 변경되는 쉐입의 높이 불일치

    미디어쿼리를 활용해 맞추려고 했으나
    다양한 PC사이즈를 감당할 수 없었음

    스크롤 이벤트시 height 값의 변동 폭을 최대한으로 줄이고, div를 감싸는 wrapper의 width값을 100%으로 고정

  • Problem 07

    중앙정렬 했음에도 센터가 맞지 않아 보이는 png 이미지

    미디어쿼리를 활용해 맞추려고 했으나
    다양한 PC사이즈를 감당할 수 없었음

    padding-right 값을 %로 지정하고, 다양한 화면 사이즈에서 봤을 때도 중앙에 있어 보이도록 크기 값을 조절

Review

Good

  • 일지를 작성한 것은 프로젝트를 마무리할 때 큰 도움이 되었다.
    처음 일지를 작성할 때 긴가민가한 마음으로 작성하였지만, 3주가 모여 큰 성과를 이루었다.
    거창한 기록이 모여 아카이브가 되는 것이 아닌 사소한 것이 모여 소중한 아카이브가 된다는 것을 느꼈다.
  • 팀원 모두 적극적으로 참여하고 의견을 냈다. 문제가 발생했을 때 외면하지 않고 모두가 그 문제를 해결하기 위해 최선을 다했다.
    비교적 순탄하게 진행된 팀 프로젝트라 생각한다.

Bad

  • 팀 프로젝트 경험이 부족해 함께 소통하며 진행하는 부분이 미숙했다.
    사람 대 사람으로 나누는 대화가 원활한가를 넘어서
    자료를 공유하고 함께 사용하는 프로그램을 능숙하게 다룰 수 있는 것이 원활한 소통이라 느꼈다.
  • 명확한 역할 분담을 하지 않고 프로젝트를 진행해 원활하지 않은 부분이 있었다.
    서로 같은 부분을 수정하거나 어떤 부분은 전혀 수정되지 않는 경우가 있었다.
    역할 분담을 명확하게 나눠 프로젝트를 진행했다면 더욱 효율적인 결과를 얻을 수 있지 않았을까 하는 아쉬움이 남는다.

Takeaway

  • 함께한다는 것은 엄청난 시너지라고 느꼈다. 나 혼자였다면 이룰 수 없을 만큼 만족스러운 결과물이 나왔다.
    함께 하는 것을 배우고, 함께가 얼마나 값진 일인지 깨달았다.
  • UIUX란 통계학이란 것을 배웠다. 그저 보기 좋은 아름다운 것이 아니다.
    수많은 경험과 실패를 통해 형성된 것이 UIUX 디자인이라는 것을 확실하게 깨달았다.
